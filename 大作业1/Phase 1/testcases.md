# 基于 MongoDB 问卷系统 - 测试用例

本项目支持完整的 UI 交互测试。请在本地运行 `app.py` 后，通过浏览器访问 `http://127.0.0.1:5000` 按照以下用例进行测试。

## 测试用例 1：用户注册与登录校验
* **前置条件**：系统正常启动。
* **测试步骤**：
  1. 在登录区输入用户名 `test_user`，密码 `123456`，点击“注册”。
  2. 再次点击“注册”。
  3. 点击“登录”。
* **预期结果**：
  1. 第一次提示“注册成功”。
  2. 第二次提示“用户名已存在”（校验拦截成功）。
  3. 登录成功，跳转到仪表盘，显示“欢迎回来, test_user”。

## 测试用例 2：动态创建问卷与逻辑注入
* **测试步骤**：
  1. 登录后点击“+ 设计新问卷”。
  2. 添加【单选题】：标题“性别”，选项“男, 女”，跳转逻辑填入 `男->q3`。
  3. 添加【文本填空】：标题“你的爱好”。
  4. 添加【数字填空】：标题“年龄”，最小数值 `18`，最大数值 `30`。
  5. 点击“正式发布”。
* **预期结果**：提示发布成功，并在仪表盘列表中出现该问卷。MongoDB `surveys` 集合中成功插入带有嵌套 `questions` 数组的文档。

## 测试用例 3：跳转逻辑触发测试 
* **前置条件**：完成用例 2，点击该问卷的“去填写”。
* **测试步骤**：在页面中，点击单选题的选项“男”。
* **预期结果**：点击瞬间，题目 2（文本填空）自动隐藏，页面只保留题目 1 和题目 3。证明前端动态解析 `jump_logic` 成功。

## 测试用例 4：后端边界值强校验 (核心考点)
* **测试步骤**：
  1. 接着用例 3，在年龄（数字填空）中输入 `15`。
  2. 点击“提交答卷”。
* **预期结果**：前端弹出后端返回的错误信息：“题目 '年龄' 不能小于 18”。系统拒绝将脏数据写入数据库。

## 测试用例 5：多选题独立拆分与均值计算统计
* **前置条件**：以合法数据成功提交 2 份答卷（例如年龄分别填入 20 和 24）。
* **测试步骤**：在仪表盘点击“看统计”。
* **预期结果**：
  1. 接口成功返回 UTF-8 编码的 JSON 数据。
  2. 数字题统计模块出现 `"平均值": 22.0`。
  3. 证明 MongoDB `$unwind` 聚合查询及 Python 后端逻辑计算正确。
